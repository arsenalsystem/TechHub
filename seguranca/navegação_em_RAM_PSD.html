Navegação em RAM (PSD)

Otimização Extrema para Usuários Linux no Fluxo Moderno de I.A. e Web

Por que RAM e não SSD?

Hoje, o navegador é o nosso ambiente de desenvolvimento e interação com I.A. (ChatGPT, Claude, VS Code Web). Essas ferramentas geram milhares de pequenos arquivos de cache e logs por segundo.

🚀 Velocidade de Resposta: A RAM é centenas de vezes mais rápida que um SSD NVMe. Isso reduz o input lag em chats de I.A. e na escrita de código.

O Profile-sync-daemon (PSD) gerencia seu perfil (extensões, cookies, cache) diretamente no tmpfs (memória RAM), sincronizando com o disco apenas quando necessário.

Instalação e Configuração Passo a Passo

1. Instale o pacote base:

sudo apt install profile-sync-daemon

2. Execute a primeira verificação (O ponto crucial):

psd parse
⚠️ Observação importante: Na primeira vez, ele dirá que o arquivo psd.conf foi criado e pede para você editá-lo. Geralmente, as configurações padrão já detectam seus navegadores automaticamente, bastando rodar o comando novamente.

3. Ative o serviço para o seu usuário:

# Habilita para iniciar junto com o computador
systemctl --user enable psd

# Inicia o serviço e move o perfil para a RAM agora
systemctl --user start psd

O que cada comando faz?

  • psd parse: Atua como um "scanner". Ele verifica quais navegadores você instalou (Chrome, Firefox, Brave) e se o arquivo de configuração está correto.
  • systemctl --user enable: Cria um link simbólico no sistema. Ele diz ao Linux: "Assim que o Rogoberto fizer login, prepare o ambiente de RAM".
  • systemctl --user start: É o gatilho de execução. Ele pega a pasta do seu perfil no disco, joga para a memória RAM e faz o navegador "acreditar" que os arquivos ainda estão no mesmo lugar.

Avaliação Técnica

✅ Prós
  • Zero latência no carregamento de abas.
  • Aumenta a vida útil do SSD (menos escritas de cache).
  • Privacidade: Cache de I.A. fica na RAM volátil.
  • Sincronização automática em segundo plano.
❌ Contras / Riscos
  • Consome memória RAM física (use se tiver 8GB ou mais).
  • Quedas bruscas de energia podem causar perda do histórico da última sessão.
  • Exige um kernel Linux estável para gerenciar o tmpfs.

Como saber se deu certo?

Para confirmar que você está navegando na velocidade da RAM, use o comando de status:

systemctl --user status psd

Procure pelo texto ● active (running). Se ele estiver lá, seu navegador agora é uma turbina!

🚀 Guia de Referência Rápida

Utilize estes comandos no terminal para gerenciar e monitorar seu Profile-sync-daemon.

1. Diagnóstico e Status

psd parse ou psd p O mais importante: Mostra quais navegadores foram detectados e se já estão na RAM.
systemctl --user status psd Verifica se o serviço está ativo. (Lembre-se: "active (exited)" significa sucesso!)

2. Gerenciamento do Serviço

# Inicia a sincronização agora
systemctl --user start psd

# Para o serviço e devolve os arquivos para o disco (segurança)
systemctl --user stop psd

# Garante que o PSD ligará sozinho com o sistema
systemctl --user enable psd

# Força uma gravação dos dados da RAM para o disco agora
psd resync

3. Configurações e Backups

Configuração: ~/.config/psd/psd.conf
Edite este arquivo se quiser escolher navegadores específicos na linha BROWSERS=(...).

Backups: ~/.config/psd/backups/
Onde o PSD guarda uma cópia de segurança antes de mover seu perfil.
