🚀 Por que RAM e não SSD?

Hoje, o navegador é o nosso ambiente de desenvolvimento principal. Ferramentas como ChatGPT, Claude e VS Code Web geram milhares de logs e arquivos de cache por segundo.

  • Velocidade: A RAM é centenas de vezes mais rápida que um NVMe, eliminando input lag.
  • Vida Útil: Reduz drasticamente o ciclo de escrita (TBW) no seu SSD físico.
  • Privacidade: Cache volátil desaparece ao desligar, mantendo o sistema limpo.

🛠️ Profile-sync-daemon (PSD)

O PSD gerencia seu perfil (extensões, cookies, cache) diretamente no tmpfs (RAM), sincronizando com o disco apenas quando necessário.

1. Instalação: sudo apt install profile-sync-daemon

2. Verificação Inicial: psd parse (Gera o arquivo de configuração).

3. Ativação:

systemctl --user enable psd
systemctl --user start psd

📊 Avaliação Técnica

✅ Prós

• Zero latência em abas.
• Sincronização em background.
• Menos aquecimento do SSD.

❌ Riscos

• Consome RAM física.
• Quedas de energia perdem a sessão atual.
• Exige Kernel estável.

🔍 Guia de Comandos Rápidos

Comando Ação
psd p Diagnóstico: Mostra o que está na RAM.
psd resync Força a gravação da RAM para o Disco agora.
systemctl --user status psd Verifica se o serviço está rodando.

🧠 Estratégia de Contexto Persistente

Como o sistema desliga após 15 minutos de inatividade, a continuidade é mantida através do "DNA da Conversa".

1. O Checkpoint

Antes de encerrar, solicite à IA: "Gere um resumo técnico (Markdown) para carregar em uma nova sessão, incluindo Decisões, Variáveis e Próximos Passos."

2. O Bootstrap (Reinicialização)

Ao abrir um novo chat, envie o comando de gatilho:

Aja como meu assistente. Assuma o contexto abaixo e ignore saudações: [COLE O CONTEÚDO AQUI]
